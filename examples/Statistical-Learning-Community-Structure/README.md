# Statistical Learning — Community Structure

Implements the behavioral protocol of Experiment 2 from Schapiro et al. (2013).
Participants are exposed to a continuous stream of abstract symbols generated by
a random walk on a graph with community structure, and are later asked to mark
perceived event boundaries in a mixed random-walk / Hamiltonian-path sequence.

---

## Graph

15 nodes arranged in 3 communities of 5 (C0 = nodes 0–4, C1 = 5–9, C2 = 10–14).
Within each community the graph is K5 minus the edge between the two boundary
nodes, giving every node exactly 4 neighbours. Cross-community edges: 4↔5, 9↔10, 14↔0.

---

## Phases

### Exposure (1,400 trials)

A continuous random walk on the graph. Stimuli are presented for 1.5 s each
with no inter-stimulus interval.

**Cover task:** press **F** if the symbol appears in its normal orientation,
**J** if it has been rotated 90°. About 20 % of symbols are rotated.
Respond on every trial.

### Parsing (600 trials)

40 alternating blocks of 15 stimuli: random-walk blocks and Hamiltonian-path
blocks. A single fixed Hamiltonian cycle is used throughout. Each Hamiltonian
block is entered at the last node of the preceding random-walk block; traversal
direction (forward / backward) is chosen randomly per block.

**Cover task:** same F / J rotation task as in the exposure phase.

**Segmentation task:** also press **SPACE** whenever you feel a natural
boundary or beginning in the sequence. Trust your intuition — there are no
right or wrong answers.

## Prerequisites

- Go 1.25+

## Running

```bash
# Fullscreen, participant 1
go run . -s 1

# Windowed (development / testing)
go run . -w -s 1
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `-s` | `0` | Participant ID (integer) |
| `-w` | off | Windowed mode (1024×768) |
| `-d N` | `-1` | Display index (-1 = primary monitor) |

---

## Output

Data are saved to `goxpy_data/` as a `.csv` file. One row per trial:

| Column | Description |
|--------|-------------|
| `phase` | `exposure` or `parsing` |
| `trial` | Trial number (1-based) |
| `node` | Graph node index (0–14) |
| `community` | Community index (0–2) |
| `block_num` | Block number within the phase |
| `block_type` | `rw` (random walk) or `ham` (Hamiltonian) |
| `pos_in_block` | Position within the block (1–15) |
| `community_transition` | `1` if the node is in a different community than the previous node |
| `is_rotated` | Whether the stimulus was rotated 90° |
| `response` | Key pressed: `F`, `J`, or `none` |
| `correct` | Whether the rotation response was correct |
| `rt_ms` | Reaction time in milliseconds |
| `space_pressed` | Whether SPACE was pressed during this trial (parsing phase) |
| `space_ms` | Time from stimulus onset to SPACE press in ms (0 if not pressed) |
